Módulos do Sistema

Apps Django especializados para cada funcionalidade

Accounts

Sistema de autenticação e gestão de usuários:

  • Registro e login de clientes
  • Perfis de usuário personalizados
  • Gestão de gráficas parceiras
  • Sistema de permissões
Products

Catálogo de produtos gráficos:

  • Categorias e tipos de produtos
  • Especificações técnicas
  • Galeria de imagens
  • Sistema de preços base
Orders

Gestão completa de pedidos:

  • Solicitação de orçamentos
  • Acompanhamento de status
  • Histórico de pedidos
  • Sistema de avaliações
Partners

Gestão de gráficas parceiras:

  • Cadastro de gráficas
  • Especialidades e capacidades
  • Sistema de reputação
  • Painel administrativo
Payments

Integração com Mercado Pago:

  • Processamento de pagamentos
  • Gestão de transações
  • Webhooks para confirmação
  • Histórico financeiro
Core

Funcionalidades centrais:

  • Página inicial e estáticas
  • Sistema de configurações
  • APIs internas
  • Ferramentas administrativas

Principais Modelos Django

Estrutura de dados para o ecossistema completo

App: Accounts
  • CustomUser - Usuário personalizado com roles
  • CustomerProfile - Perfil de clientes
  • PartnerProfile - Perfil de gráficas
App: Products
  • Category - Categorias de produtos
  • Product - Produtos com especificações
  • ProductImage - Imagens dos produtos
  • PriceRule - Regras de preço por quantidade
App: Orders
  • QuotationRequest - Solicitação de orçamento
  • Quotation - Orçamento enviado por gráficas
  • Order - Pedido formalizado
  • OrderTracking - Acompanhamento de status
  • Review - Avaliação de serviços
App: Payments
  • Transaction - Registro de transações
  • Payment - Informações de pagamento
  • Payout - Repasses para gráficas

Preparação para Deploy

Configurações e requisitos para colocar no ar

Configurações de Produção
  1. Configurar variáveis de ambiente (DEBUG, SECRET_KEY, DATABASE_URL)
  2. Configurar banco de dados PostgreSQL
  3. Configurar servidor de mídia estática (Whitenoise, AWS S3, etc.)
  4. Configurar servidor de e-mail
  5. Configurar Celery para tarefas assíncronas

Comandos para Iniciar
Iniciar projeto do zero:
# Criar ambiente virtual
python -m venv venv

# Ativar no Linux/Mac
source venv/bin/activate

# Ativar no Windows
venv\Scripts\activate

# Instalar dependências
pip install -r requirements.txt

# Criar projeto
django-admin startproject graphic_express .

# Criar apps
python manage.py startapp accounts
python manage.py startapp core
python manage.py startapp products
python manage.py startapp orders
python manage.py startapp partners
python manage.py startapp payments
Comandos de configuração:
# Migrações iniciais
python manage.py makemigrations
python manage.py migrate

# Criar superusuário
python manage.py createsuperuser

# Coletar arquivos estáticos
python manage.py collectstatic

# Testar servidor
python manage.py runserver
